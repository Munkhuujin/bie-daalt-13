**Огноо**: 2026-05-11  
**Ашигласан AI**: Claude (web), Cursor

**Тэмдэглэл**: Энэ тайлангийн анхны draft-ыг Claude-аар үүсгүүлсэн бөгөөд дараа нь өөрийн бодит ажилласан туршлагын дагуу засаж, өөрийн үгээр найруулсан. ## Танилцуулга

Энэ ажлын эхэнд би AI-г зүгээр л "хурдан код бичдэг хэрэгсэл" гэж бодож байсан. Гэтэл бараг 2 долоо хоног ажилласны эцэст үзэл бодол маань нэлээд өөр болсон. AI заримдаа надад туслахаас илүүтэйгээр будлиулсан үе бий, харин заримдаа гайхалтай хурдан үр дүн өгдөг. Хамгийн чухал нь — AI-ийн санал болгож буй бүх зүйлийг шалгахгүйгээр ашиглавал болохгүй.

Курсийн "verify, don't trust" зарчмыг лекцэд сонссон ч практик дээр учираад л өнгөн ойлголтоос гүн ойлголт болж хувирсан.

## 1. Юуг AI хийсэн, юуг өөрөө хийсэн?

### А хэсэг — Planning

Эхний үед AI-г санаа цэгцлэхэд голчлон ашигласан. Stack харьцуулалтын анхны хувилбарыг (Node.js vs Python vs Java) AI-аар гаргуулж байгаад өөрөө review хийсэн. ARCHITECTURE.md дотор Mermaid diagram-ын syntax-ийг гараар бичих гэвэл их цаг авах байсан тул AI шууд гаргаж өгсөн.

Гэхдээ Personal Task Tracker гэдэг сэдвийг сонгох, "authentication хийхгүй", "multi-user болгохгүй" гэж scope-оо хязгаарлах шийдвэрүүдийг өөрөө гаргасан. AI энд оролцоогүй — учир нь AI зүгээр л санал болгох тусам scope нь томордог. Нэг удаа AI бүр JWT auth, refresh token, middleware chain нэмэхийг санал болгосон. Энэ нь learning project гэдгийг ойлгохгүй байсан.

### Б хэсэг — Build

Express server-ийн boilerplate, SQLite connection setup гэх мэт repetitive хэсгүүдийг AI секундын дотор гаргаж өгсөн. Гараар бичсэн бол хэдэн цаг авах байсан.

Database schema-ийн анхны хувилбарыг AI гаргасан ч review-д орохоор хэд хэдэн зүйл өөрчилсөн. Жишээ нь priority дээр CHECK constraint нэмсэн, status нь enum шиг ажиллах ёстой гэж шийдсэн. AI business rule-ийг бүрэн ойлгоогүй байсан тул эцсийн schema-г өөрөө тогтоосон.

Тестүүд дээр AI scaffolding (describe, it, mock setup) сайн ажилласан. Гэхдээ AI-ийн үүсгэсэн test ихэвчлэн зөвхөн "happy path"-ыг шалгадаг байсан. Whitespace-only title, null due_date, empty body гэх мэт edge case-уудыг өөрөө нэмсэн.

### В хэсэг — Reflect

Энэ тайлангийн draft-ыг AI-аар бичүүлсэн ч жишээ бүр бодит туршлагаас гарсан. Хэт AI шиг сонсогдох хэсгүүдийг өөрөө дахин найруулж байна. Шударга хэлэхэд цаг бага үлдсэн тул бүгдийг гар бичмэл шиг хийж амжихгүй байна.

## 2. Hallucination жишээнүүд

### Жишээ 1 — PowerShell vs CMD

Setup хийж байх явцад AI Windows-ын CMD ашиглаж байгааг тооцолгүй PowerShell-ийн `New-Item -ItemType Directory` командуудыг санал болгосон. CMD дээр `'New-Item' is not recognized` гэсэн алдаа гарсан үед эхэндээ өөрийн машин дээр асуудал байна гэж бодсон. Удалгүй shell mismatch гэдгийг ойлгож, AI-д "би CMD ашиглаж байна" гэж тодорхой хэлсний дараа л `mkdir`, `type nul >` гэсэн зөв командуудыг авсан.

Энэ нь жижигхэн алдаа боловч AI миний environment-ийг автоматаар таамагладаггүйг ойлгуулсан.

### Жишээ 2 — Route ordering bug

Хамгийн их толгой эргэлдсэн алдаа энэ байсан. F4 (due date views) хэрэгжүүлсний дараа `/api/tasks/views/overdue` дуудахад 404 буцсан. AI эхэндээ "файлаа хадгалсан уу", "server restart хийсэн үү" гэх ерөнхий зөвлөгөө өгч байсан. Гэтэл асуудал нь route ordering байсан — `router.get('/:id', ...)` route нь `/views/overdue`-ийг өмнө барьж аваад `id = "views"` гэж буруу таниж байсан.

Үүнийг олохын тулд taskRoutes.js файлыг мөр мөрөөр өөрөө уншиж байж олсон. Засвар нь `/views/*` route-уудыг `/:id`-ийн өмнө байрлуулсан. Express-ийн "first match wins" зарчмыг практик дээр л ойлгосон.

### Жишээ 3 — SQLite connection pooling

Нэг үед AI "SQLite-д connection pool ашиглах ёстой" гэж санал болгосон. Гэхдээ SQLite single-file database учраас pool бараг ач холбогдолгүй. AI том системд хэрэглэдэг pattern-уудыг жижиг project-д хүчээр зөвлөдөг — "best practice" нэрээр overengineering хийдэг.

## 3. Security асуудал

Энэ нь хамгийн их сургамжтай хэсэг байсан.

F2 (Search/Filter) хийх явцад AI дараах байдалтай код өгсөн:

```javascript
query += ` ORDER BY ${sortBy} ${sortOrder}`;
```

Энд `sortBy` бол хэрэглэгчээс шууд ирж буй query string. Хэрэв `?sort_by=title; DROP TABLE tasks; --` гэж оруулбал — database устгана. Чухал нь AI parameterized query-г бусад газарт ашигласан ч энэ нэг газарт ашиглаагүй байсан.

Шалтгаан нь SQL-ийн `?` placeholder нь VALUE-уудад ажилладаг, column name-уудад ажилладаггүй. Тиймээс whitelist аргаар хамгаалах ёстой:

```javascript
const allowedSorts = ['created_at', 'due_date', 'priority', 'title'];
const safeSortBy = allowedSorts.includes(sortBy) ? sortBy : 'created_at';
```

AI энэ ялгааг автоматаар тооцдоггүй. Хүний review энд орлуулашгүй. "Ажилладаг код" нь "аюулгүй код" гэсэн утга биш.

## 4. AI хамгийн их хэрэгтэй байсан үе

**Boilerplate generation** — Express server, middleware setup, repository pattern. Эдгээрийг бичих ажил гар мэхээр зөвхөн.

**Documentation** — OpenAPI YAML-ийг гараар бичих гэвэл indentation алдаа, syntax алдаа их гарна. AI цэвэр гаргаж өгсөн.

**Mermaid diagram** — Syntax-ийг сурахын тулд documentation унших гэж зуу зуун цаг алдахаас сэргийлсэн.

**Conventional Commits** — Format-ыг тогтмол хадгалахад тус болсон.

## 5. AI саад болсон үе

**Overengineering** — Нэг удаа AI logging framework, dependency injection, custom error architecture нэмэхийг санал болгосон. Бүгдийг дагасан бол өнөөдөр turn-in амжуулахгүй байсан.

**Generic debugging** — Route ordering bug дээр "restart хий" гэх ерөнхий зөвлөгөө 15 минут алдсан. Specific context өгмөгц л зөв хариу гарсан.

**Хэт урт хариу** — 2 мөрийн асуултанд 30 мөрийн хариу гардаг үе байсан. Уншиж амжихгүй.

## 6. Skill atrophy эрсдэл

Хамгийн их зовоосон асуудал энэ. AI-аар бүгдийг хийлгэвэл өөрөө syntax-ыг мартдаг. Шалгалт AI-гүй болохыг мэдэх тул өөртөө дараах дүрэм тавьсан:

- **curl тестүүд гар бичмэлээр** — Бүх 20+ запросыг өөрөө бичсэн. cmd-д escape character яаж ажиллахыг өөрөө сурах хэрэгтэй болсон.
- **Bug-ийн root cause-ыг эхлээд өөрөө хайх** — AI-аас шууд "засах" гэж асуухгүй. Эхлээд log, stack trace, кодоо уншина.
- **AI-ийн кодыг copy-paste биш** — Уншиж ойлгож байж commit хийсэн.
- **CLAUDE.md-г өөрөө шинэчлэх** — Project convention-уудыг гараар бичсэн нь codebase-ийг хэр ойлгож байгааг шалгах байсан.

Шударга хүлээн зөвшөөрөхөд — Jest mock API, complex SQL JOIN, Mermaid syntax зэргийг одоо AI-гүйгээр бичиж чадахгүй. Энэ нь дасгал хэрэгтэй хэсгүүд. Нуухгүй, тэмдэглэх ёстой бодит зүйл.

## Дүгнэлт

Энэ assignment миний хувьд "AI бол хурдан кодлогч" гэсэн ойлголтыг өөрчилсөн. Одоо AI-г junior developer шиг ажилладаг хамтрагч мэт хардаг — заримдаа маш ухаалаг санал өгдөг, заримдаа гайхалтай буруу зүйл хэлдэг. Бүх зүйлийг шалгахгүйгээр итгэх боломжгүй.

Хамгийн чухал өөрчлөлт — "код бичих" ажил надад одоо хамгийн чухал ажил санагдахаа больсон. Spec тодорхой бичих, AI-д зөв context өгөх, гарсан үр дүнг security өнцгөөс үзэх — эдгээр нь илүү чухал болсон.

REST API хийсэн нь энэ ажлын гол үнэ цэн биш. Гол үнэ цэн нь AI-тай хэрхэн зөв ажиллахыг сурсан явдал юм. Энэ нь ирээдүйд хаана ажиллахдаа хэрэгтэй чадвар болно гэж бодож байна.
