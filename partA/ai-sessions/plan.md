# AI Session 1 — Project Planning

**Огноо:** 2026-05-04

**Хэрэглэсэн tool:** Claude (web)

**Зорилго:** Төслийн сэдэв сонгох, хамрах хүрээг (scope) тогтоох, болон PROJECT.md файлын эхний бэлтгэлийг гаргах.

## Гол асуултууд ба шийдвэр

1. Сэдэв сонголт: Personal task tracker сэдвийг сонгосон. Учир нь энэ нь өгөгдлийн бүтэц болон логик харилцааг (Labels, Filtering) бүрэн харуулах боломжтой. Мөн өдөр тутмын хэрэглээтэй, ойлгомжтой тул feature-үүдийг тодорхойлоход амар байсан.

2. Stack: Node.js + Express-ийг сонгосон. AI-тай харьцуулахад хөгжүүлэлт хурдан, setup бага, мөн ийм жижиг хэмжээний API хийхэд тохиромжтой гэж үзсэн.

3. Feature тоо: Нийт 3 үндсэн feature (CRUD + Label + Search/filter) хийхээр төлөвлөсөн.

4. Төслийн хэлбэр: Хэтэрхий нүсэр биш, зааварт заасан хамгийн бага шаардлага болох REST API + minimal frontend бүтэцтэй байна.

## AI-аас гарсан гол санал

1. AI-аас зөвхөн нэг feature (жишээ нь: зөвхөн Task CRUD) хийх нь бие даалтад fail болох эрсдэлтэйг анхааруулж, хамрах хүрээгээ өргөжүүлэхийг санал болгосон.

2. Сонгосон 3 feature-т зориулж API endpoint-уудын жагсаалтыг (GET, POST, PUT, DELETE) гаргаж өгсөн.

3. Төслийн үндсэн баримт бичиг болох PROJECT.md-ийн ерөнхий бүтцийг (draft) боловсруулж өгөв.

## Өөрөө хийх ёстой зүйлс

- PROJECT.md-ийн "Зорилго" хэсгийг AI-аар бичүүлэх биш, яг яагаад энэ төслийг хийж байгаа, юу сурахыг хүсэж байгаагаа өөрийн үгээр найруулан бичнэ.

- Төслийг заасан хугацаанд (2 долоо хоног) амжуулахын тулд юуг хийхгүй (Out of scope) байхыг нэмж тодорхойлох.

- Feature бүрийг яг юу хийхийг илүү тодорхой (ambiguity багатай) болгож бичих

## AI-н алдаа эсвэл сэжигтэй санал

Repository setup-ын явцад AI нь миний орчныг Windows Command Prompt шалгалгүй Powershell коммандуудыг (`New-Item -ItemType Directory`) санал болгосон. Эдгээр командыг ажиллуулахад "'New-Item' is not recognized as an internal or external command" гэсэн алдаа гарсан. Энэхүү алдааг би cmd-д тохирсон `mkdir`, `type nul >` командуудыг ашиглаж зассан. AI-аас командын санал авахдаа ямар орчин дээр ажиллаж байгаагаа тодорхой хэлж байх ёстойг цаашид анхаарах ёстойг сануулсан.
